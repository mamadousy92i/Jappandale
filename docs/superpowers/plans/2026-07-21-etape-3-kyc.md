# Plan d'implémentation — Étape 3 : KYC (vérification d'identité)

**Goal:** Permettre à un utilisateur de soumettre ses pièces d'identité, à l'administrateur de les valider/rejeter manuellement, et refléter le statut KYC dans l'espace compte. Prérequis pour la création de campagnes (étape 4).

**Architecture:** Nouvelle app Django `apps.kyc` (modèle `KycDocument`). Le modèle `User` gagne un champ `kyc_status` et des champs d'audit de revue. Endpoints `/api/kyc/`. Validation manuelle via l'admin Django (journalisée). Frontend : section KYC active dans l'espace compte (upload multipart).

## Global Constraints
- Backend port 8001. Commits français sans « Co-Authored-By ». Aucune mention « YAMB International ».
- Charte or/blanc, Playfair + Inter. Interface française.
- TDD backend. Fichiers média servis en dev uniquement.
- Statuts KYC : `NON_SOUMIS` (défaut), `EN_ATTENTE`, `VALIDE`, `REJETE`.

## Modèle de données
- `User.kyc_status` (CharField choices, défaut NON_SOUMIS), `kyc_reviewed_at` (nullable), `kyc_review_note` (blank), `kyc_reviewed_by` (FK self/admin, nullable).
- `KycDocument` : `user` (FK, related_name="kyc_documents"), `document_type` (CNI / PASSEPORT / JUSTIFICATIF_ACTIVITE), `file` (FileField upload_to="kyc/%Y/%m/"), `uploaded_at`.

## Endpoints
- `POST /api/kyc/submit/` (authentifié, multipart) : champs `document_type` + `file`. Crée un `KycDocument` ; si le statut est NON_SOUMIS ou REJETE, passe à EN_ATTENTE. Retourne `{ kyc_status, documents }`.
- `GET /api/kyc/` (authentifié) : `{ kyc_status, documents: [...] }`.
- `kyc_status` ajouté à la réponse de `GET /api/auth/me/`.
- Admin : actions « Valider le KYC » / « Rejeter le KYC » sur les utilisateurs (renseignent kyc_status, kyc_reviewed_by, kyc_reviewed_at) — journalisation via l'admin log.

## Frontend
- Type `User.kyc_status`.
- `apiFetch` : ne pas forcer `Content-Type: application/json` si le corps est un `FormData` (support multipart).
- Espace compte, section « Vérification d'identité » pilotée par `kyc_status` :
  - NON_SOUMIS / REJETE → formulaire d'upload (type de pièce + fichier).
  - EN_ATTENTE → message « dossier en cours de vérification ».
  - VALIDE → badge « Identité vérifiée ».

## Tâches
1. Backend : champs KYC sur User + app `apps.kyc` + modèle `KycDocument` + migration + admin (actions valider/rejeter) + média en dev.
2. Backend : endpoints `/api/kyc/submit/` et `/api/kyc/` + `kyc_status` dans `/api/auth/me/` + tests.
3. Frontend : support multipart dans `apiFetch`, type `User.kyc_status`, section KYC de l'espace compte + vérification de bout en bout.
